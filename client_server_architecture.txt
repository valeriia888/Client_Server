Сервер- поставщик услуг
Клиент - заказчик услуг
Клиент и сервер расположены на разных машинах и взаимодействуют между собой 
с помощью протоколов (например http)
Есть двух уровневая архитектура ( клиент + сервер)
Есть трехуровневая (клиент+ сервер+база данных)
Сервер может быть клиентом, в случае, когда он отправляет запрос в базу 
данных.

Клиент отправляет http запрос (request) на сервер - сервер обрабатывет и 
отправляет ответ обратно (http response)

Самый распространенный клиент это браузер, также это может быть приложение
в телефоне

Есть тонкий и толстый клиент
Тонкий - компьтера или программа, переносит большую часть своих задач по обра
ботке на сервер (браузер)
Толстый - обрабатывают информацию на основе своих возможностей (например 1 с)


Веб сайт - носит информационный характер, состоятоят из веб страниц, просто 
предоставляют контент для юзера и не предоставляют возможность взаимодействия
пользователь не может загружать свою инфу
(например сайты с рецептами, с музыкой)

Веб приложение - позволяют юзеру ввводить получать и манипулировать данными 
с помощью взаимодествия (отправляют запросы на сервер) ( фейсбук, ютуб, твитер
 соц сети, амазон, переводчики, конвертеры валют)
 
Веб сервис - это api
