API - Application programming interface
Это контракт, по которому системы взаимодействуют между собой. 
В контаркте указано : ко мне можно обращаться так-то и так-то, я обязуюсь делать это и то"

API - может быть:
Local  - локальный, взаимодействие между компонентами одной системы
Remote - удаленный- позволяет взаимодействовать разным системам

Например где мы используем API:
1. Когда хотим интегрировать платежную систему в наш сайт, заказ билетов в кинотеатре 
тре - мы вибираем ряд, время, дату, сам фильм и как только переходим к оплате
те - начинается работа API 
2. Когда мы к сайте подкручиваем онлайн-консультанта
3. Когда мы ходим зарегистрироваться через любую социальную сеть
4. Когда на сайте по заказу пиццы, есть множетсво полей, которые пользователь
должен заполнить, чтобы получить пиццу, через API  мы подкручиваем стороню 
систему, которая будет выдавать подсказки и пользователю будет
быстрее и проще все заполнить

Способы вызова API : 
- вызов функции системой ( наш локальный API ) 
- вызов метода другой системы 
- вызов метода человеком (например нам надо залить большое количество 
тестовых данных в систему  и чтобы этого не делать на графическом интерфейсе дол
го - мы можем воспользоваться API - мы открываем постман пишем http request 
с помощью метода post  заливаем туда данные , которые нужно указать например в поле регистрации и запускаем , после этого запрос запускается на сервер и мы получаем необходимые данные.
- через графический узер интерфейс

Как работает API :
- визов самой функции (методы get. post)
- входные данные (http request)
- выходные данные (http response)


Ендпоинты  это часть юрл, точки входа, адрес на который отправляются 
сообщения. Шлюз которые соединяет работу сервира и внешнего интерфейса
